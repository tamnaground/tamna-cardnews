# 결과물 저장 규칙 (Output Location Rules)

클로드(Claude)가 이 프로젝트에서 생성하는 **모든 결과물**(카드뉴스 이미지, 문서, 데이터 파일 등)은 반드시 아래 규칙에 따라 저장한다.

## 저장 위치

1. **로컬 Windows 환경에서 작업할 때** (경로 `C:\Users\hepyh`가 존재하는 경우):
   - 모든 결과물을 `C:\Users\hepyh\claude` 폴더 아래에 저장한다.
   - 프로젝트별로 하위 폴더를 만들어 정리한다. 예: `C:\Users\hepyh\claude\tamna-cardnews\`
   - 폴더가 없으면 먼저 생성한다.

2. **원격/클라우드 환경에서 작업할 때** (이 리포지토리를 통해 작업하는 경우):
   - 모든 결과물을 리포지토리의 `outputs/` 폴더 안에 저장한다.
   - 리포지토리 루트나 다른 위치에 결과물 파일을 흩어놓지 않는다.
   - 작업 완료 후 커밋하고 푸시하여, 사용자가 로컬 PC(`C:\Users\hepyh\claude` 아래)에서 pull 받을 수 있게 한다.

## 파일 정리 규칙

- 결과물 파일명은 내용을 알 수 있게 짓는다. 예: `outputs/2026-07-06_제주여행_card_1.png`
- 같은 주제의 카드뉴스 여러 장은 같은 접두어를 사용해 묶는다.
- 임시 파일이나 중간 산출물은 커밋하지 않는다.
