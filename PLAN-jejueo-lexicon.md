# PLAN: 검증된 제주어 어휘 사전 단일화 (`데이터/제주어_어휘.json`)

> 우선순위: **4위**
> 이유: 콘텐츠의 신뢰성이 곧 이 채널의 존재 이유인데, "검증된 제주어" 목록이 현재 3곳에 중복 분산돼 있다 — `.claude/skills/제주어영상/SKILL.md`(25~30행), `첨부자료/실사영상_제작패키지.md`(14~15행), `샘플영상/build_audio.py`(대사 안). 한 곳에서 어휘를 수정하면 나머지가 낡은 채 남고, 새 콘텐츠를 만들 때마다 출처를 다시 뒤져야 한다. 출처가 붙은 단일 어휘 데이터 파일은 카드뉴스 생성기(PLAN-cardnews-generator)와 에피소드 작성(PLAN-episode-data) 양쪽의 입력이 된다.

## 목표

지금까지 검증해 사용한 모든 제주어 어휘·문형을 출처와 함께 담은 기계가독(JSON) 사전을 만들고, SKILL.md가 개별 어휘를 나열하는 대신 이 파일을 참조하게 한다. 이후 새 어휘는 반드시 이 파일에 출처와 함께 등록한 뒤에만 콘텐츠에 사용한다는 규칙을 SKILL.md에 명문화한다.

## 수정/생성할 정확한 파일

| 파일 | 작업 |
|---|---|
| `데이터/제주어_어휘.json` | **신규 생성** — 어휘 사전 본체 |
| `데이터/README.md` | **신규 생성** — 스키마 설명과 등록 규칙 (10줄 내외) |
| `.claude/skills/제주어영상/SKILL.md` | **수정** — 2단계(21~29행)의 어휘 나열을 사전 파일 참조로 교체. 단, 진행상/청유 문형 요약과 "실존 인물 목소리 클로닝 금지" 문구는 그대로 유지 |
| `첨부자료/실사영상_제작패키지.md` | **수정** — 14~15행 어휘 나열 뒤에 "전체 어휘: 데이터/제주어_어휘.json" 참조 1줄 추가 (기존 내용 삭제하지 않음) |

## 단계별 작업 순서

### 1단계: 기존 어휘 전수 수집

다음 파일에서 제주어 표현을 전부 추출한다:
- `.claude/skills/제주어영상/SKILL.md` 25~27행: 하영, 무사, 기여, 혼저, 고장, 질, 어신디, 게메마씸, 폭싹 속았수다, 혼저옵서예, -암져/-엄져, -암신고, -읍주
- `샘플영상/build_audio.py` LINES: 오람져, 오람신고, 어신디, 났져, 밭디, 들카부덴, 게메마씸, 그치민, 피키여, 안네, 들어갑주
- 루트 card_1~5.png (Read로 열어 카드에 실린 단어 확인 — 최소 '고장' 포함)

### 2단계: 스키마 확정 및 JSON 작성

```json
{
  "version": 1,
  "sources": {
    "namu_grammar": "https://namu.wiki/w/제주 방언/문법",
    "jeju_dict": "https://www.jeju.go.kr/culture/dialect/dictionary.htm",
    "grandculture": "https://jeju.grandculture.net/jeju/toc/GC00702008",
    "bazaar_blues": "https://www.harpersbazaar.co.kr/article/66891"
  },
  "entries": [
    {
      "jeju": "고장", "std": "꽃", "type": "명사",
      "example_jeju": "비 그치민 고장 하영 피키여.",
      "example_std": "비 그치면 꽃이 많이 필 거야.",
      "sources": ["jeju_dict"], "verified": true,
      "used_in": ["card_1.png", "샘플영상/제주삼춘_비오는날_대화.mp4"]
    },
    {
      "jeju": "-암져/-엄져", "std": "~하고 있네 (진행상)", "type": "어미",
      "example_jeju": "비 하영 오람져.", "example_std": "비가 많이 오네.",
      "sources": ["namu_grammar"], "verified": true, "used_in": []
    }
  ]
}
```

필드 규칙: `type`은 `명사|동사|형용사|부사|감탄사|어미|관용구` 중 하나. `verified`는 sources에 근거가 1개 이상 있을 때만 true. 출처가 아직 없는 후보 어휘는 `verified: false`로 넣되 콘텐츠 사용 금지.

### 3단계: 검증 스크립트 없이 검증 가능하게

별도 검증 코드는 만들지 않는다 (과잉 엔지니어링). 대신 `데이터/README.md`에 검증 원라이너를 적어 둔다:

```bash
python3 -c "
import json; d = json.load(open('데이터/제주어_어휘.json'))
bad = [e['jeju'] for e in d['entries'] if e['verified'] and not e['sources']]
assert not bad, f'출처 없는 verified 항목: {bad}'
print(len(d['entries']), '개 항목 OK')"
```

### 4단계: SKILL.md·실사영상_제작패키지.md 참조 교체

SKILL.md 2단계의 어휘 나열(26행)을 다음으로 교체:

> 어휘·예문은 `데이터/제주어_어휘.json`에서 가져온다 (`verified: true`인 항목만 사용). 새 어휘가 필요하면 WebSearch로 출처(제주도청 사전, 나무위키 문법, 디지털제주문화대전)를 확인하고 사전에 등록한 뒤 사용한다.

문형 요약(-암져/-엄져, -읍주 등 25·27행)과 29행의 목소리 클로닝 금지 조항은 **삭제하지 않고 유지**한다.

### 5단계: 커밋

`데이터/` 디렉토리와 수정된 md 2건을 커밋.

## 성능이 낮은 모델이 놓치기 쉬운 엣지 케이스

1. **표기 변이 문제**: 제주어는 표준 표기법이 정착되지 않아 같은 말이 여러 표기로 나타난다 (예: 오람져/오람쪄, 게메마씸/게메마씀). 항목을 임의로 "맞는 표기"로 통일하지 말고, 대표 표기를 `jeju`에 두고 `variants: ["..."]` 필드에 이형을 기록할 것. 어느 쪽이 대표인지는 제주도청 사전 표기를 우선한다.
2. **옛한글(아래아 ㆍ) 인코딩**: 제주어 표기에는 아래아(ᄋᆞ, 예: ᄒᆞᆫ저)가 쓰이기도 한다. 파일은 UTF-8(BOM 없음)로 저장하고, 현대 한글 표기(혼저)를 `jeju`로, 아래아 표기는 `variants`로 넣는다 — 렌더링 파이프라인의 NotoSansKR 폰트가 옛한글 조합을 지원하지 않을 수 있기 때문이다.
3. **어미와 단어의 혼동**: `-암져`는 단어가 아니라 어미다. 어미 항목의 `jeju` 값에 하이픈을 유지하고(`"-암져/-엄져"`), 카드뉴스 생성기가 이 항목을 단어 카드로 뽑지 않도록 `type: "어미"`로 구분해 둘 것.
4. **문장 통째 등록**: "폭싹 속았수다", "혼저옵서예"는 단어가 아니라 관용구다. `type: "관용구"`로 넣고 뜻풀이를 문장 번역으로 쓸 것 ("수고하셨습니다", "어서 오세요").
5. **출처 링크 부패**: 나무위키·기사 링크는 변할 수 있다. sources를 URL 직접 나열이 아니라 최상위 `sources` 맵의 키 참조로 설계한 이유가 이것이다 — URL이 바뀌면 한 곳만 고친다. 이 구조를 "단순화"한다고 항목마다 URL을 복사해 넣지 말 것.
6. **SKILL.md 과잉 삭제**: 2단계를 통째로 사전 참조 한 줄로 바꿔 버리면, 문형 활용 규칙(진행상·청유)과 윤리 조항(목소리 클로닝 금지)이 사라진다. 교체 범위는 **어휘 나열 부분만**이다.

## 완료 기준 (사용자가 직접 검증)

1. `python3 -m json.tool 데이터/제주어_어휘.json > /dev/null && echo OK` — JSON 문법 통과.
2. 3단계의 검증 원라이너가 "N개 항목 OK"를 출력한다 (N ≥ 15: 기존 콘텐츠에 등장한 어휘가 전부 수집됐다면 최소 15개).
3. 모든 `verified: true` 항목에 `sources`가 1개 이상 있고, 모든 항목에 `example_jeju`/`example_std` 쌍이 있다.
4. `grep -n "제주어_어휘.json" .claude/skills/제주어영상/SKILL.md` 결과가 있다 (스킬이 사전을 참조함).
5. SKILL.md에 "클로닝" 문구가 여전히 존재한다: `grep -c "클로닝" .claude/skills/제주어영상/SKILL.md` ≥ 1.
6. 기존 영상 대사(`샘플영상/build_audio.py` 또는 episodes/rainy_day.json)에 쓰인 제주어 표현이 전부 사전에서 검색된다.
